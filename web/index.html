CS184 Final Project

Feature Summary

The goals of this project were to implement different algorithms to solve inverse kinematics problems, provide a variety of different joints to choose from, and explore the space of constraint satisfaction within inverse kinematics.

The technical details for what I am about to explain below are outlined in this tech report.

The methods which I implemented were:

  • Pseudoinverse
  • Damped Least Squares
  • Generalized Jacobian Transpose
  • Gradient Descent (Did not achieve desirable results)

The robot was a linked robot. An assumption was made that the links of the robot would form a tree (a node was not allowed to have more than one parent). This assumption was used to simplify the calculation of the Jacobian. The following joint types were implemented:

  • Fixed Axis
  • Ball and Socket
  • Telescoping (translational)

I also explored playing around with constraints. The generalized jacobian transpose method was implemented in order to handle hard constraints (C(x) = 0). I implemented positional constraints, which can be toggled by clicking on a joint (examples below). I also attempted to explore inequality constraints (C(x) > 0), which required the formulation of the problem as a non-linear optimization problem. My first attempt at solving this was to use gradient descent, but ultimately the method was too simplistic and solutions did not converge when constraints were "added" (via lagrange multipliers). I did not have enough time to try more advanced methods such as BFGS or Levenberg-Marquardt.

Finally, a key-frame animation system was implemented. Every time the a key is pressed, a snapshot of the robot's current configuration is stashed away. The user can replay this animation at any time and can even adjust the frame rate at which this animation plays back.

In the examples below, only the generalized jacobian transpose method is shown, because it is capable of handling constraints. As mentioned above, I was unable to get gradient descent to converge nicely when using lagrange multipliers (I was able to get it to play nicely without constraints, but that was not interesting since I already had several other methods which worked fine without constraints).
